Contacts

User Story: List of Contacts

  • As a user I'll be able to see a list of contacts

AC: acceptance criteria

  • The title page should be "Contacts", in orange and capitalized.
  • The list contains Contact Name, Email and Actions that can be done by contact.
  • When there are none contacts the following message “There are none contacts” should be displayed

User Story: Order contacts

  • As a user I'll be able to order the list of contacts

AC: acceptance criteria

  • The list can be ordered by Name or Email ascending or descending. Rows next to each title should be displayed.g
  • The contacts are alphabetically ordered by name by default.

User Story: New Contact

  • As a user I'll be able to add new contacts.

AC: acceptance criteria

  • By clicking on New Contact button, 2 fields are going to be displayed: Name and E-mail.
  • Save and Cancel buttons are displayed at the end of the screen.
  • Name field is required and empty by default. When field is saved empty, the following validation should be displayed: ‘The contact name is required’.
  • Also, Name field is a text with a max length of 35 characters. When field is saved with more than 35 characters, the following validation is displayed: ‘The contact name cannot have more than 35 characters’
  • Email format is validated, the following message should be displayed when does not meet the criteria: "The email format is not correct".
  • Email should be unique for the contacts. There cannot be 2 contacts with the same Email. The following validation should be displayed when entering a user with an existing Email: ‘A contact with this email already exists’.
  • If the user clicks on ‘New Contact’ without saving the previous contact a message should be displayed: "Are you sure that you want to leave without save?" with Accept and Cancel buttons."
  • ‘Save’ button should run all validations.
  • When all required data is completed and validated, by clicking on ‘Save’ button, the contact is going to be saved
  • By clicking on ‘Cancel’ button, New contact section is closed and the contact is not saved.
  • When Contact is saved, it should be added to the list, with the options to edit and delete.

User Story: Edit Contact

  • As a user I'll be able to edit a contact

AC: acceptance criteria

  • The contact can be edited by clicking on Edit button (Pencil).
  • Contact information should be displayed to be edited.
  • Save and Cancel buttons are displayed at the end of the screen.
  • Name is required and has a max length of 35 characters.
  • When Name field is saved empty, the following validation should be displayed: ‘The contact name is required’.
  • When Name field is saved with more than 35 characters, the following validation is displayed: ‘The contact name cannot have more than 35 characters’
  • Email is required and has max length of 35 characters.
  • Email format is validated, the following message should be displayed when does not meet the criteria: "The email format is not correct".
  • Email should be unique for the contacts. There cannot be 2 contacts with the same Email. The following validation should be displayed when entering a user with an existing Email: ‘A contact with this email already exists’.
  • When all required data is completed and validated, by clicking on ‘Save’ button, the contact is going to be saved with the modifications.
  • By clicking on ‘Cancel’ button, Edit contact section is closed and the contact is not saved.

User Story: Delete Contact

  • As a user I'll be able to delete a contact

AC: acceptance criteria

  • The contact can be deleted by clicking the delete button (red x)
  • A confirmation message should be displayed: 'Are you sure that you want to delete ?'
  • The user is able to cancel the delete.
  • When the user accepts the deletion, the contact should be deleted from the list of contacts.
  • When the user clicks on Cancel button, contact should remain on the screen.
