Epic: Contacts

User Story: List of Contacts

  • As a user I'll be able to see a list of contacts

AC: acceptance criteria

  • The title page should be "Contacts"
  • The list contains Contact Name, Email and actions that can be done by contact
  • When there are none contact the following message “There are none contact” is displayed

User Story: Order contacs

  • As a user I'll be able to order the list of contacts

AC: acceptance criteria

  • The list can be ordered by Name or Email, ascending or descending
  • The contacts are alphabetically ordered by name by default

User Story: New Contact

  • As a user I'll be able to add a new contacts

AC: acceptance criteria

  • The Name is requiered
  • The Name is empty by default
  • The Name is a text with a max lenght of 35 characters
  • The Email is requiered
  • The Email is empty by default
  • The Email is a text with a max lenght of 35 characters
  • The Email format is validated, the following message should be displayed: "The email format is not correct"
    • See here email format
  • The Email should be unique for the contacts
  • When save the contact, should be added to the list, with the options to edit and delete

User Story: Edit Contact

  • As a user I'll be able to edit a contact

AC: acceptance criteria

  • The contact can be edited by clicking the edit button
  • The contact information should be displayed to be edited
  • The Name is requiered
  • The Name is a text with a max lenght of 35 characters
  • The Email is requiered
  • The Email is a text with a max lenght of 35 characters
  • The Email format is validated, the following message should be displayed: "The email format is not correct"
    • Email format: http://tools.ietf.org/html/rfc5322#section-3.4.1
  • The Email should be unique for the contacts

User Story: Delete Contact

  • As a user I'll be able to delete a contact

AC: acceptance criteria

  • The contact can be edited by clicking the delete button
  • A confirmation message should be displayed: "Are you sure that you want to delete "
  • The user is able to cancel the delete
  • When the user accept the deletion, the contact should be deleted from the list of contacts
